### 人力资源市场基本要求

<br>

#### 1.1备案信息

基本功能描述：

<emsp>录入和修改人力资源市场用户详细信息的内容。保存后上报地市、省备案。

数据说明：



| 数据项         | 必填 | 说明                                 |
| -------------- | :--- | :----------------------------------- |
| 人力资源市场名 | 是   | 显示人力资源市场名称（唯一）         |
| 所属地区       | 是   | 显示所属地市、市县、区域（不可修改） |
| 联系人         | 是   | 显示联系人姓名（支持中文及英文）     |
| 联系电话       | 是   | 格式必须符合（区号）+电话            |
| 传真           | 否   | 格式必须符合（区号）+电话            |

<br>

#### 1.2数据填报

基本功能描述：

<emsp>人力资源市场用户填报当期采集数据。根据预先设定的模版在规定范围内填报，填写完成后保存，确认无误后上报。支持查询（根据调查期查询填报信息）、保存（暂时保存填报信息，但不会上报，保存后还可以修改）、上报（上报后数据不能再修改）等操作。

### 市（区县）基本要求

<br>

#### 1.1用户管理

基本功能描述：

<emsp>录入用户名、密码、人力资源市场用户基本信息等内容。可以对录入的人力资源市场用户的信息进行修改、删除。但有上报数据的人力资源市场用户不能删除。

<br/>

数据说明：

| 数据项               | 必填 | 说明                                          |
| -------------------- | ---- | --------------------------------------------- |
| 用户名               | 是   | 字母、数据和'-'、'_'，禁止使用中文或？%等字符 |
| 密码                 | 是   | 同上                                          |
| 确认密码             | 是   | 必须与密码一致                                |
| 人力资源市场用户编码 | 是   |                                               |
| 人力资源市场用户名称 | 是   |                                               |
| 所属地区             | 是   | 自动根据用户所属地区默认                      |
| 行业                 | 是   | 两级下拉选择                                  |

<br>

#### 1.2上报备案

基本功能描述：

​	<emsp>对已经填写完整备案信息的企业进行备案操作

<br>

#### 1.3用户填报管理

基本功能描述：

​	<emsp>查看并且审核人力资源市场用户上报的建档期和调查期数据

<br>

#### 1.4报表管理

基本功能描述：

​	<emsp>已经填好的报表上报给省，并且可以查看往期的报表数据

要求：

​	<emsp>定义好接口，交接方案（等）

<br>

#### 1.5数据管理

基本功能描述：

​	<emsp>对本市的账号进行查询，以及监测点数据分析（仅能分析本市监测点数据）

要求：

​	<emsp>可进行条件查询，数据分析最好能够做出表格展示

<br>

#### 1.6发布通知

基本功能描述：

​	<emsp>发布、删除通知信息。

相关权限描述：

​	<emsp>市（区县），省

<br>

##### 1.6.1列表：

​	<emsp>列出当前用户发布的所有通知信息。列表项包括：标题、发布时间。

<br>

##### 1.6.2新增：

​	<emsp>新增一条通知信息。数据项如下：

|  数据项  | 必填 |     说明     |
| :------: | :--: | :----------: |
| 通知标题 |  是  |   50字以内   |
| 通知内容 |  是  |  2000字以内  |
| 发布时间 |  否  | 系统自动生成 |
| 发布单位 |  否  | 系统自动生成 |

<br>

##### 1.6.3修改：

​	<emsp>选择一条通知，进行修改。数据项同新增

<br>

##### 1.6.4删除：

​	<emsp>删除一条通知信息

<br>

#### 1.7浏览通知

基本功能描述：

​	<emsp>浏览查看通知信息



<br>

<br>

### 省基本要求

<br>

#### 1.1用户备案

基本功能描述：

​	<emsp>查看所有已经备案人力资源市场用户的详细信息，但不可以更改

<br>

#### 1.2报表管理

基本功能描述：

​	<emsp>审核地市（区县）上报的数据

<br>

#### 1.3数据汇总

基本功能描述：

​	<emsp>查看人力资源市场用户的汇总数据

<br>

#### 1.4取样分析

基本功能描述：

​	<emsp>可根据不同需要显示各市（区县）人力资源市场用户的数量和占比

<br>

#### 1.5固定报表

基本功能描述：

​	<emsp>查看各调查期内各市（区县）人力资源市场用户需求人数、需求比重、环比和同比的变化情况。

<br>

#### 1.6图表分析

<br>

##### 1.6.1对比分析：

基本功能描述：

​	<emsp>使用折线图、表格形式对比两个调查期的人力资源市场数据变动情况

<br>

##### 1.6.2趋势分析：

基本功能描述：

​	<emsp>使用折线图、表格形式展示多个连续调查期内的需求人数、求职人数的变动情况

<br>

#### 1.7数据查询与导出

基本功能描述：

​	<emsp>对全省已创建账号进行条件查询

<br>

#### 1.8发布通知

基本功能描述：

​	<emsp>发布、删除通知信息

| 数据项   | 必填 | 说明         |
| :------- | :--- | :----------- |
| 通知标题 | 是   | 50字以内     |
| 通知内容 | 是   | 2000字以内   |
| 发布时间 | 否   | 系统自动生成 |
| 发布单位 | 否   | 系统自动生成 |

<br>

#### 1.9浏览通知

基本功能描述：

​	<emsp>浏览查看通知信息

<br>

#### 1.10系统管理

基本功能描述：

​	<emsp>系统管理相关功能

<br>

##### 1.10.1上报时限

基本功能描述：

​	<emsp>新增或修改调查期

<br>

##### 1.10.2用户管理

基本功能描述：

​	<emsp>建立省和地市的用户

<br>

##### 1.10.3角色管理

基本功能描述：

​	<emsp>根据需要建立多种角色，不同角色对应不同功能。系统预定义一些角色

<br>

##### 1.10.4系统监控

基本功能描述：

​	<emsp>查看当前系统工作情况。包括CPU、内存、硬盘等信息和应用系统的一些信息
